An operating system for news

Beams is an open development framework for news sites, emphasizing both security and speed. In fact, our tests show it offers an unparalleled combination of both compared to the server software most major news sites now run in production.

Beams' features include:

  • Secure server using Nginx and Node.js. A simple, fast server stack to get your project up and running quickly using the secure HTTPS protocol. Plus, it uses the latest development tools to do it, so you can upgrade and maintain solid performance at the same time.
  • jQuery 2.1.4. The framework includes a full self-hosted copy of this popular JavaScript library so your project can quickly call functions from it from the client side.
  • Ultraresponsive layout. The Beams layout is based on a 20-column, 4,000-pixel grid. This allows elegant resizing of elements on any size screen, from a large TV down to a smartphone. Video players are also responsive, including support for embedded players for YouTube, Vimeo, and self-hosted HTML5 video.

You can download the latest version of Beams from us here, or if you prefer, it's also available via GitHub, where you can fork it, file bug reports, or contribute code back to make Beams even better.

Quick. What do all these news orgs have in common?

Answer: None of their websites run as fast or securely as this one. That's because we run on Beams, and they don't.

Yet.

We recently benchmarked our site server against those of the most popular English-language news sites. None of the above organizations' servers handled batches of 1,000 requests as fast as ours did running Beams, and none used the encrypted HTTPS protocol to protect their site visitors' privacy, as Beams does by default.

Beams' secret weapons for speed are modern server technologies like Nginx and Node.js -- tools that many news organizations have yet to adopt fully.

Take a look a look at Beams today to see what it can do for your site. It's open source under MIT license, so you can download the framework today and begin innovating yourself. Or, if you prefer, drop the Roscoe Labs team an email today to see how we might be able to help you implement Beams.

Why HTTPS for news?

Among major U.S. news sites, there is currently almost zero use of the HTTPS protocol, which encrypts communication between publishers and site visitors.

In an era when the NSA, foreign governments, and sundry garden-variety fraudsters are looking to hack anyone they can, this urgently needs to change. Just ask the New York Times.

On its developer blog in November 2014, the Times issued a friendly "HTTPS Challenge" to the entire news industry to convert to HTTPS by the end of 2015.

At Roscoe Labs, we've met the NYT's challenge -- and then some. We've not only converted our own site to HTTPS, but we've provided other publishers with an open tool to do the same, via the Beams framework.

Please, let's all get started protecting users today. Together.

Covering the client side too

Don't let all the server-side awesomeness fool you. Since Day One, Beams has been built intentionally, proudly, as a full-stack framework, including handy client-side tools as well.

Beams includes a 20-column ultraresponsive grid to make your site display well on any screen, from a TV down to a smartphone.

We've also included a self-hosted version of the latest version of jQuery, so you can call on that popular JavaScript library easily, right from the user's browser.
